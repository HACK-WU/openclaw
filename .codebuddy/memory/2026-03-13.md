## 修复: CLI Agent 创建时 bridge.json 写入失败

**问题**: 前端 `createCliAgent()` 通过 `agents.files.set` RPC 写入 `bridge.json` 时，后端拒绝该文件名（`unsupported file "bridge.json"`）。

**原因**: `src/gateway/server-methods/agents.ts` 中的 `ALLOWED_FILE_NAMES` 白名单只包含 bootstrap 文件和 memory 文件，不包含 `bridge.json`。

**修复**:

1. `src/agents/workspace.ts`: 添加 `DEFAULT_BRIDGE_FILENAME = "bridge.json"` 常量
2. `src/gateway/server-methods/agents.ts`: 导入 `DEFAULT_BRIDGE_FILENAME` 并加入 `ALLOWED_FILE_NAMES` 白名单

## 修复: 创建群聊时 projectDirectory.trim() 报错 TypeError

**问题**: 从侧边栏 `+` 按钮创建群聊时，`group-chat.ts:1471` 报 `Cannot read properties of undefined (reading 'trim')`。

## 设计文档: CLI Agent 独立管理

创建了 `docs/design/group-chat-bridge/cli-agent-management.md`，定义 CLI Agent 与通用 Agent 的完全隔离方案：

- 独立存储：`cli-agents/bridge.json` 作为全局注册表，集中管理所有 CLI Agent 元信息（不写入 openclaw.json）
- 独立目录：`~/.openclaw-dev/cli-agents/{agentId}/`，子目录仅存放个性化文件（IDENTITY.md + AGENTS.md）
- 独立 API：`cliAgents.*` RPC 命名空间
- 独立 UI：CLI Agent 只展示 Overview / Files / Test 三个 Tab
- 测试功能：创建时和管理页面均支持 CLI 测试启动（弹框 xterm.js 终端）
- 群聊触发：@CLI Agent 时走 PTY 路径而非 LLM 推理

**架构调整记录**：bridge.json 从各子目录移到 cli-agents/ 根目录，作为统一注册表，避免遍历子目录读取困难。

## 设计文档补充: Agent 名称与 AgentID 规范（§7）

在 `cli-agent-management.md` 中新增 §7 章节，定义 Agent 名称与 AgentID 的职责边界（适用于通用 Agent + CLI Agent）：

- **Agent 名称**：用户可见的显示名，所有 UI 位置展示名称，允许中文
- **AgentID**：系统内部标识符，仅限 `[a-zA-Z0-9_]`，用于数据传输/存储/目录名
- **Overview 是唯一展示 AgentID 的地方**，其他 UI 都只展示名称
- 创建时默认 AgentID = 名称，含中文时需手动指定
- **通用 Agent 创建也需支持手动指定 AgentID**（当前缺失）
- 新增 P0 实施阶段：名称/ID 规范（AgentID 校验函数 + 前端双字段交互）
