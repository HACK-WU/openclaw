# 2026-03-09

## 群聊工具卡片修复 — 对齐单聊实现

### 问题

群聊中工具卡片渲染逻辑与单聊存在多个差异：

1. 后台只有 `onToolStart`（phase: start），没有 result 阶段 → 工具卡片永远显示"运行中"
2. 前端没有传 `onOpenSidebar` 给工具卡片 → 无法在侧边栏查看工具输出
3. 流式阶段工具卡片位置在气泡内部 vs 单聊在气泡外部

### 修改的文件

1. **`src/auto-reply/reply/agent-runner-execution.ts`**: 扩展 `onAgentEvent` 中的 `onToolStart` 回调，使其也转发 `phase: "result"` 事件（之前只转发 start/update）。对于 result 阶段不触发 typing indicator。
2. **`src/group-chat/agent-trigger.ts`**:
   - 新增 `addToolResult()` 函数处理工具完成事件
   - 扩展 `onToolStart` handler 处理 start/result 两个阶段
   - result 阶段时添加 `role: "tool"` 消息到 collector 并立即广播
3. **`ui/src/ui/views/group-chat.ts`**:
   - `GroupChatViewProps` 新增 `onOpenSidebar?` 属性
   - `renderGroupMessage` 和 `renderGroupStreamBubble` 添加 `onOpenSidebar` 参数
   - 所有 `renderInlineToolCards` 调用传入 `onOpenSidebar`
   - 工具卡片从气泡内部移到气泡外部（与单聊一致）
4. **`ui/src/ui/app-render.ts`**: 群聊 props 中传入 `onOpenSidebar: state.handleOpenSidebar`
