## 详细组件分析

### 设备配对与令牌管理

- 配对流程
  - 请求：创建待处理请求并记录设备公钥、角色、作用域等元信息
  - 批准：合并角色与作用域，为指定角色生成新令牌；令牌轮换策略确保重连后生效
  - 拒绝：清理待处理请求
  - 元数据更新：支持动态合并角色与作用域
- 令牌校验
  - 校验设备已配对、角色存在、令牌未吊销、机密比较安全相等、请求作用域被授权
  - 使用原子锁保证并发一致性
- 令牌生命周期
  - 生成：按需生成新令牌
  - 轮换：支持按角色轮换并同步更新作用域
  - 吊销：标记撤销时间戳，后续校验失败

```mermaid
flowchart TD
Start(["开始"]) --> Req["创建待处理请求"]
Req --> Approve{"批准?"}
Approve --> |是| Issue["为角色生成新令牌"]
Approve --> |否| Reject["删除待处理请求"]
Issue --> Use["客户端使用令牌调用"]
Use --> Verify{"校验令牌与作用域"}
Verify --> |通过| OK["成功"]
Verify --> |失败| Fail["失败"]
Reject --> End(["结束"])
OK --> End
Fail --> End
```

### Origin 检查机制（跨域与 CSRF 防护）

- 核心逻辑
  - 解析 Origin，标准化 host/hostname
  - 允许列表命中优先
  - 与请求 Host 匹配
  - 回环地址放宽（Origin 与请求 Host 均为回环）
- 防护效果
  - 阻止跨域脚本发起请求
  - 降低 CSRF 风险

HostMatch --> |否| Loopback{"Origin/Host 均为回环?"}

### 安全配置指南与最佳实践

- 配置与凭据

  - 避免在命令行历史中暴露密钥
- 日志与审计
  - 启用敏感信息脱敏；避免将日志文件暴露给其他用户
  - 审计器检测文件权限问题与通道配置风险
- 网络与绑定
  - 绑定策略选择：回环、局域网、Tailnet、自动、自定义
  - 控制 UI 仅在受保护网络或启用鉴权时开放
- 通道与访问控制
  - DM/群组策略：默认“配对”，可切换为白名单或禁用
  - 严格限制“开放”策略，避免任意发送者进入

## 故障排查指南

- 常见问题定位
  - 设备未配对：令牌校验返回“设备未配对”
  - 角色缺失：令牌校验返回“角色缺失”
  - 令牌不匹配：安全机密比较失败
  - 作用域不足：请求作用域超出授权范围
  - Origin 不允许：浏览器 Origin 检查失败
  - TLS 指纹不匹配：客户端拒绝握手
- 审计与诊断
  - 使用安全审计命令查看严重性汇总与修复建议
  - 检查文件权限、日志级别、通道策略
